# arch-sync: sync

Synchronize system context from all sibling repos and regenerate ARCHITECTURE.md for the current repo.

## What this command does

1. Reads `arch-sync.json` from the current working directory
2. For each configured repo: reads the specified documentation files
3. Copies those files to `.claude/system-context/<repo-name>/`
4. Generates (or regenerates) `ARCHITECTURE.md` at the project root

## Step-by-step instructions

### Step 1: Read the config

Read `arch-sync.json` from the current working directory. If it does not exist, stop and tell the user to run `/arch-sync:init` first.

Parse the JSON to get:
- `repos` array with entries containing: `name`, `path`, `files` (optional, defaults to `["CLAUDE.md", "INTEGRATION.md", "README.md"]`)
- Optional `currentRepo` object with `name` and `description` for the current repo (falls back to the directory name)

### Step 2: Read docs from each repo

For each repo in `repos`:

1. Resolve the `path` field relative to the current working directory
2. For each file in `files` (or defaults `["CLAUDE.md", "INTEGRATION.md", "README.md"]`):
   - Attempt to read `<resolved-path>/<filename>`
   - If the file does not exist: note it as "not found" — do not error
3. Track what was found and what was missing per repo

If `--repo <name>` argument was provided, only process the named repo (still regenerate ARCHITECTURE.md with all previously cached context).

### Step 3: Copy files to local context folder

Unless `--no-copy` flag is provided:

For each repo, for each file that was successfully read:
1. Create directory `.claude/system-context/<repo-name>/` if it does not exist (use Bash `mkdir -p`)
2. Write the file content to `.claude/system-context/<repo-name>/<filename>`

### Step 4: Generate ARCHITECTURE.md

Use the system-architecture skill to write `ARCHITECTURE.md` at the project root. The file should describe the entire system from **this repo's perspective**.

Use this structure:

```markdown
# Architecture — <current-repo-name>

> Generated by arch-sync. Run `/arch-sync:sync` to regenerate.
> Last updated: <current date>

## Overview

<2-3 sentence description of what this repo does and its role in the system>

## System Components

| Component | Role | Location |
|---|---|---|
| <repo-name> | <one-line role> | <path> |
...

## Inbound Dependencies — Who Depends on This Repo

<List each component that uses/calls/reads this repo. For each:>
- **<Component>**: What it gets from us. Which API/protocol/file format. Auth method.

If nothing depends on this repo, write "This is a leaf component — no other system component depends on it."

## Outbound Dependencies — What This Repo Depends On

<List each component this repo calls or reads. For each:>
- **<Component>**: What we get from it. Which API/protocol/file format. Auth method.

If this repo has no external dependencies within the system, write "This is a root component with no inbound system dependencies."

## End-to-End Data Flows

Describe 2-5 key flows that pass through this repo. Use arrows to show the path.

**Flow: <name>**
```
<Component A> → <this repo> → <Component B> → result
```
<Brief description of what data moves, what transforms happen here>

## Integration Contracts

Things the whole system agrees on that affect this repo:

### Auth & Credentials
<How auth works at the boundaries: tokens, vault, OAuth, etc.>

### Shared Data Formats
<Message schemas, state shapes, event formats used across repos>

### Protocols & Transport
<SSE, REST, gRPC, LangGraph Platform API, etc.>

<Vars that must be consistent across repos: Supabase URLs, MCP URLs, etc.>
```

### Step 5: Print sync report

After completing, print a summary like:

```
✅ arch-sync complete

Repos synced:
  ✅ mcp           — 3 files copied  (.claude/system-context/mcp/)
  ✅ langgraph      — 2 files copied  (.claude/system-context/langgraph/)
  ⚠️  ui            — 1 file missing  (INTEGRATION.md not found)

Generated: ARCHITECTURE.md
```

## Argument handling

- `--force`: Overwrite existing `.claude/system-context/` files even if unchanged
- `--repo <name>`: Only re-sync one specific repo (all others use cached context for ARCHITECTURE.md generation)
- `--no-copy`: Regenerate ARCHITECTURE.md from already-cached context without re-reading source repos

## Error handling

- `arch-sync.json` not found → stop with: "No arch-sync.json found. Run /arch-sync:init to set up."
- Repo path does not exist → note as warning in report, continue
- File not readable → note as warning in report, continue
- Never fail silently — always show what was skipped and why

## Tips

- Run this command whenever you add a new repo or significantly change another repo's CLAUDE.md
- The generated ARCHITECTURE.md is designed to be committed — it gives Claude full system context without re-reading all repos on every session
- To refresh a single repo after its docs changed: `/arch-sync:sync --repo mcp`
- **ARCHITECTURE.md is a generated file — never edit it directly.** Only `/arch-sync:sync` is allowed to write or overwrite it. Humans and Claude (in normal sessions) must not modify it manually. It will be overwritten on the next sync. If you discover something wrong or missing, update the relevant source doc (`CLAUDE.md` or `INTEGRATION.md` in the appropriate repo) and re-run sync. If you find a discrepancy between docs and actual code, add a `⚠️` note to the source doc, not to ARCHITECTURE.md itself.
